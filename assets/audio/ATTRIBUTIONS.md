# Audio attributions

Every shipped sound file in `assets/audio/` and the rules under which it
may be redistributed.

## Synthesized in-repo (no third-party rights)

Generated by `tools/gen_audio.py` from this repository's own code. No
external samples were used. Treat as part of the source — the project
license applies.

| File | Description |
| --- | --- |
| `sfx/snap.wav` | 60 ms exponentially decaying noise burst at low amplitude. |
| `sfx/chime.wav` | C5–E5–G5 sine-tone arpeggio with overlap and exponential decay (~600 ms total). |

Run `python3 tools/gen_audio.py` from the repo root to regenerate. The
script is deterministic (seeded RNG for the snap), so re-runs reproduce
the exact same bytes.

## Silent stubs (per-letter, per-word)

`letter/<L>.wav` (26 files) and `word/<WORD>.wav` (41 files) ship as
80 ms of digital silence. These are placeholders pending real Hungarian
pronunciation recordings — see DESIGN.md §7 ("Recording responsibility:
the user is Hungarian; recording falls to them").

To replace a stub, drop a same-named `.wav` file into the same path. No
code changes are needed; the audio service URLs are derived from the
letter/word identifier at runtime.

When real recordings land, add their attribution under a new section
below (recording person + date + license). The expected v1 license is
the project's own (the user records their own voice).

## Adding third-party samples in the future

Any sample taken from outside this repository must be:

- CC0, CC-BY, or compatibly-licensed for redistribution under our
  project license.
- Added here with: filename, source URL, license, attribution string
  (if required), and date checked.
- Copied into `assets/audio/` (or a clearly-named subdirectory) before
  bundling.

Avoid samples that require attribution we can't legibly surface to a
6-year-old user. Prefer CC0.
